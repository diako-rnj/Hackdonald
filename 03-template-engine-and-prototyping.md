# Template Engine & Prototyping

## Process Overview
Once the context is mapped, the **Frontend & Design Agent** takes over to generate the visual and functional aspect of the product. The system uses standard and widely adopted web technologies (e.g., React or Next.js) to build out the front end.

## The Template Pool
The system maintains a centralized repository of pre-designed, modular layouts called the **Template Pool**. These templates are categorized by industry (Food, Healthcare, Beauty, etc.) and feature complexity.

### Subscription Filtering
The number of templates presented to the agent for compilation depends entirely on the customer's subscription tier:
- The agent filters the Template Pool based on the `context.json`.
- It selects the optimal base design.

## Prototyping Phase
After selecting the template, the agent begins the compilation process:

1. **Context Injection:** Colors, fonts, and imagery provided in the contract are injected into the template's CSS/theme files.
2. **Component Assembly:** Dynamic components (like contact forms, product galleries, or booking calendars) are mounted based on the requested features.
3. **API Wiring:** The frontend components are wired up to the endpoints generated by the Backend Agent.

## Final Exhibition
The result of this process is a "full functional website." The codebase is bundled, deployed to a staging environment, and a prototype URL is generated. This prototype is then ready for customer review and exhibition, completing the website generation phase.
